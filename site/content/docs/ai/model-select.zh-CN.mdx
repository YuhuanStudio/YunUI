import { ModelSelect } from "yunui/ai";

可搜索、可分组的模型选择器。它是完全受控的——由你持有 `value`（以及可选的 `pinned` 列表），并通过 `onChange` / `onTogglePin` 更新它们。每个选项都是基于插槽的 `ModelSelectOption`，因此图标、能力图示、徽章与详情行都由宿主渲染，保持与框架无关。

## 导入

```tsx
import { ModelSelect } from "@yuhuanowo/yunui/ai";
```

## 基础用法

将选中的 id 保存在 state 中，并通过 `onChange` 回传。固定（pin）是可选的，且以相同方式受控——提供 `pinned` 与 `onTogglePin`。触发器宽度由 `className` 决定（默认为 `w-64`）。

## 选项

每一项都是一个 `ModelSelectOption`。除了 `id`、`label` 和 `group` 之外，其余都是可选插槽，让宿主可以渲染所需的任何内容：

- `groupLabel` / `groupIcon` —— 该选项所属分组显示的标题与图标。
- `searchText` —— 折入搜索匹配的额外文本（别名、原始 id）。
- `icon` —— 行首图示，通常是 `<ModelIcon>` 或 `<ProviderIcon>`。
- `badges` —— 行尾的能力图示（如 `<CapabilityIcon>`）与状态标记。
- `detail` —— 标签下方的次要行，用于 id、别名或规格。
- `meta` —— 对齐到行末的最右侧插槽，可放置任意内容。
- `disabled` —— 渲染该选项但禁止选择。

能力图示来自 YunUI 共享的能力配置（`CapabilityIcon`），因此选择器、筛选标签与独立徽章在图标或颜色上不会发生偏差。

## 筛选与固定

将 `filters` 作为 `ModelSelectFilter[]` 传入——每项包含 `key`、`node`（标签）、可选的 `title` 以及对每个选项运行的 `match` 谓词。`pinned` 与 `onTogglePin` 会在顶部呈现一个固定分组，`renderFooter` 接收可见数量。通过 `labels`（placeholder、search、all、pinned、noResults 等）可覆盖任意内置文案。
